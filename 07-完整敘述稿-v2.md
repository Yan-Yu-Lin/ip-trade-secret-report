## 第一段：開場破題（Speaker 1，約 1.5 分鐘）

### 一個問題

如果全世界都知道晶片怎麼設計、電視怎麼做聲音、自駕車怎麼用光達看路 — 為什麼不是每家公司都能做得跟台積電、Dolby、Waymo 一樣好？

答案很簡單：**因為「知道怎麼做」和「真的做得出來」之間，有一道巨大的鴻溝。** 那道鴻溝裡的東西 — 製程參數、良率調校、設備配方、測試 know-how、失敗經驗資料庫 — 就是營業秘密。

### 核心主張

專利保護的是「你發明了什麼」— 它是公開的、有期限的（20 年），誰都可以去讀。
營業秘密保護的是「你怎麼把它做出來」— 它是不公開的、沒有期限的，但一旦外洩，法律保護和商業價值可能大幅崩塌。

用一句話說：

> **專利是公開的邊界線。營業秘密是私有的執行系統。**

今天我們要講的，就是這個「執行系統」怎麼變成一門生意 — 怎麼靠它賺錢、怎麼保護它、以及當它被偷走的時候會發生什麼事。

## 第二段：什麼是營業秘密？專利 vs 營業秘密（Speaker 1，約 1.5 分鐘）

### 營業秘密的法律定義

首先要澄清一件事：**不是公司自己說是秘密就算營業秘密。** 在法律上，營業秘密必須同時滿足三個條件：

1. **不為公眾所知** — 不是一般人能輕易取得的資訊
2. **具有經濟價值** — 正因為它是秘密的，所以才有商業價值
3. **公司採取了合理的保密措施** — 有實際運作的控制機制，不只是蓋個「機密」章

這三個條件缺一不可。台灣的營業秘密法和美國的 DTSA（Defend Trade Secrets Act）都有類似的定義。後面我們會看到，第三個條件 —「合理保密措施」— 在實際案例中是最常被檢驗的。

### 專利和營業秘密的根本差異

專利的本質是一個交易：你**公開**你的發明細節，換取 20 年的排他權。全世界都可以去讀你的專利。20 年後，任何人都可以合法使用。

營業秘密走完全相反的路：你**不公開**，靠保密來維持競爭優勢。只要秘密不洩漏，保護可以永遠持續。但代價是：你沒有排他權 — 如果別人自己獨立發明了同樣的東西，或者合法地逆向工程你的產品，你無法阻止他們。

舉個最極端的例子：可口可樂的配方從 1886 年至今超過 130 年，從未申請專利。如果當初申請了，20 年後全世界都可以合法仿製。

### 決策矩陣：什麼時候該選哪個？

真正的問題不是「專利還是秘密」，而是：**「如果這個技術被公開了，會怎樣？」**

| 因素 | 傾向專利 | 傾向營業秘密 |
|-----|--------|----------|
| 逆向工程難度 | 產品賣出去就能被拆解 | 製程藏在工廠裡，外部看不到 |
| 生命週期 | 20 年內就會被淘汰 | 價值可能超過 20 年 |
| 侵權偵測 | 從產品就能發現侵權 | 濫用發生在競爭對手工廠，極難舉證 |
| 獨立發明風險 | 競爭對手很可能自己想到 | 沒有昂貴實驗很難重新發現 |

### 混合策略

真正厲害的公司不是選一邊，而是兩個都用。重點是：**專利保護別人能觀察到的層面，營業秘密保護實作層的 know-how。** 這不是「同一個東西又申請專利又當秘密」，而是同一個技術的不同層面用不同方式保護。

台積電就是最好的例子。它有超過 11.4 萬件專利申請 — 但同時還有 **475,462 筆內部登錄的營業秘密**（2013-2024，這是台積電自己的內部登錄系統，不是政府註冊）。

- **專利**保護可觀察的架構：電晶體結構、封裝方式、量測方法
- **營業秘密**保護實作層：製程窗口（溫度、壓力、氣體比例、時序）、設備調校配方、良率學習資料庫

這裡有一個重要洞見：**量產 know-how 通常是「負面知識」— 經過上千次失敗後知道什麼不能做，往往比最終配方本身更有價值。** 你不只要重新發明，還要重新經歷所有失敗。

## 第三段：Know-How 怎麼變成錢？（Speaker 2，約 3 分鐘）

### 價值轉化的三條路

營業秘密可以透過三種路徑變成現金流：

1. **溢價 (Premium)** — 因為你的良率比別人高、品質比別人好，同樣的產品你可以賣更貴或成本更低。台積電走的是這條路。
2. **授權抽成 (Licensing)** — 把 know-how 打包成授權組合，向使用者收取授權金。Dolby 走的是這條路。
3. **成本/風險降減** — 透過保密來防止技術流失，降低被追趕的風險。這是所有擁有營業秘密的公司都在做的基本功。

### 授權包怎麼設計？

如果你走授權路線，有一個根本矛盾要先解決：

> **買家要先看到秘密才知道值不值得買，但看了秘密就不是秘密了。**

這叫做「驗貨悖論」。解決方法是：**把授權設計成分階段的產品，先賣信心，再賣秘密。**

五個階段：
1. **NDA + 評估包**：高階流程圖、去敏感化的測試數據 — 讓買家知道效果，但不知道怎麼做到的
2. **付費試用**：有限的配方範圍和試產協議 — 開始收錢
3. **簽約授權**：完整 SOP、設備設定、材料規格
4. **驗收里程碑**：買家證明能重現效能後，交出進階參數
5. **商業量產**：故障排除資料庫、缺陷庫、下一代改進

每個階段搭配不同的付款結構：

| 付款類型 | 對應的是什麼 |
|--------|----------|
| 前期授權金 | 取得機密資料包的權利 + 回收研發成本 |
| 里程碑付款 | 證明被授權方能重現效能 |
| 按量/營收抽成 | 分享生產銷售的上行利潤 |
| 最低授權金 | 防止拿到技術後擱置不用 |
| 技術支援費 | 工程師培訓、問題排除、現場調試 |
| 稽核權 | 確認被授權方有沒有少報 |

### 授權合約的關鍵條款

除了付款結構，一份完整的 know-how 授權合約還需要規範：

| 條款 | 為什麼重要 |
|-----|----------|
| 授權範圍（field of use / 地區 / 專屬 vs 非專屬） | 限定被授權方能用這個技術做什麼、在哪裡做 |
| 改良技術歸屬（improvements / grant-back） | 被授權方在使用過程中做出的改進歸誰？ |
| 再授權（sublicense） | 被授權方能不能把技術再轉授給第三方？ |
| 終止後義務 | 合約結束後必須返還、銷毀資料，停止使用，保密義務持續 |
| 洩密後救濟 | 禁制令、損害賠償、刑事風險 |

### BMC 商業模式圖

BMC（Business Model Canvas，商業模式圖）是一個九宮格的分析框架，用來描述一家公司怎麼創造、傳遞、獲取價值。營業秘密不只出現在其中一格 — 它滲透了整個九宮格：

| BMC 區塊 | 營業秘密的角色 |
|---------|------------|
| 關鍵資源 | 製程配方、隱性 know-how、良率數據 |
| 價值主張 | 更快投產、更高良率、更低失敗風險 |
| 收入來源 | 授權金、技術支援費、認證費 |
| 關鍵活動 | 保護、改良、培訓、稽核 |
| 客戶關係 | 長期技術支援、改進共享 |
| 成本結構 | 安全、法律、監控、訴訟準備 |

跟專利型 BMC 最大的差異：

> **專利 BMC：「沒有我的許可你不能合法做這件事。」**
> **營業秘密 BMC：「你可以試，但我能讓你更快、更便宜、更少損失地做到。」**

專利變現的是排除權。營業秘密變現的是執行優勢。

## 第四段：案例 — Dolby，最乾淨的授權機器（Speaker 3，約 3 分鐘）

### Dolby 是什麼？

Dolby Laboratories 是一家幾乎不做硬體的公司。FY2025 總營收 $1.349B（約台幣 430 億），其中**約 92.5% 的營收來自授權** — 也就是 $1.248B 是別人付的授權金。全公司只有約 2,200 名員工，卻管理著約 1,000 家被授權方。

許多高階電視、手機、劇院系統和串流內容鏈，都因為支援 Dolby 技術而進入了 Dolby 的授權生態系。

### 它怎麼做到的？

Dolby 的授權包不只是「專利許可」— 它是五層 IP 的組合：

1. **專利** — 數百項全球專利（音訊壓縮、Dolby Atmos、Dolby Vision HDR）
2. **Know-How** — 實作所需的專有資訊（營業秘密、設計、規格）
3. **Show-How** — Dolby 提供的教學、技能傳授、測試結果、技術示範
4. **軟體** — 編譯後的參考解碼器（禁止逆向工程、禁止修改）
5. **品牌** — Dolby 商標和雙 D 標誌

Dolby 用一個雙層制來確保收到每台設備的授權金：
- **晶片商**（Qualcomm、MediaTek）把 Dolby 技術做進晶片，但只能賣給 Dolby 認可的授權方
- **設備商**（Samsung、LG、Apple）做終端產品，每台出貨付授權金

如果你不是 Dolby 的合格授權方，你連有 Dolby 技術的晶片都買不到。

### 為什麼大家願意付？

技術上可以不用 Dolby。但商業上極為困難。

首先是**標準綁定**：Dolby Digital (AC-3) 寫進了 DVD、ATSC 數位電視、Blu-ray 的規格。要做符合這些標準的設備就必須有 Dolby 授權。

其次是**生態系鎖定**：主要串流平台（Netflix、Amazon、Disney+、Apple TV+）大量採用 Dolby 格式輸出內容。內容創作者用 Dolby 工具製作 → 平台用 Dolby 格式輸出 → 消費者期待 Dolby 體驗 → 製造商必須授權。這是一個自我強化的循環。

### 稽核怎麼保護營收？

Dolby 的授權合約有一個厲害的機制：**交叉驗證**。Dolby 同時從晶片供應商和系統授權方收季度報告，兩組數據比對就能發現少報。

少報超過應付金額 5%？構成重大違約 — 後果是**三倍未付授權金 + 利息 + 稽核法律費用全額**。蓄意少報？Dolby 可以直接通知晶片供應商斷供。

2024 年 Roku 拒絕讓 Dolby 行使稽核權，結果被告上法院，Dolby 在德國取得了初步禁制令。

### 三重護城河

即使 Dolby 所有專利明天都過期，競爭對手仍然需要：
- **營業秘密層**（專有軟體 + know-how + show-how）→ 無到期日
- **品牌層**（Dolby 商標和雙 D 標誌）→ 無到期日
- **認證層**（每個產品上市前都要通過 Dolby 測試）→ 信任基礎

專利是三道護城河中唯一有到期日的那一道。

## 第五段：案例 — 台積電，最強的模式是不授權（Speaker 4，約 3 分鐘）

### 技術授權的另一面

技術授權不只是「要不要授權」的問題，也包括「什麼東西絕對不能授權」。台積電展示的是授權策略的邊界 — 有些 know-how 最好的變現方式不是授權出去，而是自己用它來賣服務。

台積電 2024 年營收 US$90.08B（約台幣 2.9 兆），幫 522 家客戶生產 11,878 種產品。先進製程（7nm 以下）佔營收的 69%。

### 為什麼不授權？

台積電的 know-how 不是一個配方，而是一個**系統**：

- 精確的製程窗口（每一步的溫度、壓力、氣體、時間）
- 良率學習資料庫（什麼條件下良率會崩潰）
- 針對每台機器的設備調校配方
- 缺陷診斷（哪些訊號預示問題）
- 供應商管理（哪個化學品批次會造成變異）
- 十幾年累積的負面知識

把這些授權出去等於同時：失去秘密 + 創造競爭對手。

所以台積電選了另一條路：**把秘密留著，賣的是「我幫你做」的服務。** Apple、Nvidia、AMD 付的是代工費，但實際上買的是台積電的秘密製程能力帶來的：更高良率（更低成本）、更快投產（更早上市）、更可靠的量產（更低風險）。

這是營業秘密 BMC 最極致的展現：**你可以試著自己做，但我能做得更好、更快、更便宜。**

## 第六段：案例 — Waymo v. Uber，當秘密被偷走（Speaker 5，約 3 分鐘）

### 時間壓縮攻擊

營業秘密竊盜的本質是**時間壓縮攻擊** — 偷的人買的不是資訊，是跳過數年研發的速度。

Anthony Levandowski 是 Google 自駕車計畫（Waymo）的核心工程師，專門做 LiDAR — 讓自駕車「看見」周圍環境的光達感測器。當時第三方商用 LiDAR 一顆要 $75,000 以上，Waymo 自研的更便宜更好。

2015 年 12 月，Levandowski 離職前六週，在公司筆電上安裝專用軟體，從 Waymo 內部設計伺服器下載了 **14,107 份檔案（9.7 GB）** — 包括 LiDAR 電路板藍圖、製造規格、供應商名單、原始碼、測試文件。然後他把檔案搬到 USB 硬碟，**擦掉整台筆電**（重灌 Linux 覆蓋 Windows）試圖銷毀證據。

2016 年他成立 Otto，被 Uber 以約 $680M 收購，成為 Uber 自駕部門主管。

### 怎麼被抓的？

2016/12/13，Waymo 員工被**意外 CC** 進一封來自 LiDAR 零件供應商的信件，標題「OTTO FILES」，附件是 Otto/Uber 的 LiDAR 電路板設計圖 — 跟 Waymo 的機密設計驚人地相似。

Waymo 啟動調查。Google 的安全工程師用三個內部工具還原了真相：Armada 確認下載來源、Bit9 記錄了 USB 連接、GRR 在筆電被擦掉之前就已經把存取日誌傳回伺服器。即使筆電被銷毀，日誌都還在。

### Uber 在收購前已看到大量紅旗

Uber 收購 Otto 前請了鑑識公司 Stroz Friedberg 做盡職調查。報告揭露：Levandowski 離職後仍存取 Google 檔案、個人電腦有 50,000 封工作信件、銷毀了含機密的光碟、在調查進行中嘗試刪除證據 — 甚至在鑑識公司辦公室裡嘗試清空垃圾桶。

但 Uber 照樣完成收購。Stroz 報告從未提交給 Uber 董事會。

法官 William Alsup：

> **「這是我見過最大的營業秘密犯罪案件。規模不是小的，是巨大的。」**

> **「數十億美元的未來利益在眼前，當那種財務誘因存在時，好人也會做出可怕的事。」**

### 結果

- 民事和解：Uber 給 Waymo **0.34% 股權（~$245M）** + 承諾不使用 Waymo 機密
- 刑事：Levandowski 被起訴 33 項竊盜罪，認罪 1 項，判 18 個月
- **Trump 在 2021 年任期最後一天特赦了他** — 從未入獄服刑
- Uber 最終賣掉整個自駕部門，前後投入近 $3B
- 聯邦檢察官 David Anderson：**「偷竊不是創新。」**

## 第七段：策略建議與防禦分析（Speaker 6，約 3 分鐘）

### 內控不只是防禦 — 它是變現的基礎建設

很多人把內控當作「防止壞事發生」。但更重要的是：

> **如果你不能證明有「合理保密措施」，在法律上你的秘密根本不算營業秘密。**

台灣最高法院 102 台上 235 號判決的核心：蓋「機密」章不夠，你需要有**實際運作的控制機制**。沒有合理保密措施，你告不了偷你東西的人，授權合約也沒有法律效力。

### 五層控制

| 層級 | 具體措施 |
|-----|--------|
| **法律層** | NDA、發明歸屬條款、保密存續條款、稽核權 |
| **員工層** | 角色式存取、培訓紀錄、離職面談、歸還/刪除認證 |
| **技術層** | Need-to-know 權限、DLP（資料外洩防護）、浮水印、日誌、禁止批量匯出 |
| **組織層** | 營業秘密登錄冊、揭露審批流程、供應商管控 |
| **證據層** | 不可竄改日誌、版本控管、存取歷史、簽署的交接紀錄 |

### 三個案例的內控教訓

**Waymo 教訓 — 離職是最危險的時刻**
Levandowski 在離職前六週就開始大量下載，但沒有觸發即時警報。應該要有：離職觸發的自動存取稽核、USB 管控、異常下載量告警。

**台積電/TEL 教訓 — 保護面必須延伸到供應鏈和實體世界**
台積電廠區有 DLP、無手機區、存取日誌 — 對數位外洩很有效。但陳力銘用的是人際關係 + 手機拍照：找到現職工程師讓他們開檔案，然後拍螢幕。DLP 擋得住下載和轉寄，擋不住有人對著螢幕按快門。遠端工作更讓機密離開了無手機區的保護。2026 年一審判決：陳力銘 10 年、TEL 台灣罰金 NT$1.5 億。

**Dolby 教訓 — 稽核是營收保護機制**
雙層報告交叉驗證讓少報無所遁形。少報 5% 以上就要賠三倍。拒絕稽核直接被告。

### 護城河 = 對手繞過你的成本

> **替代成本 = 研發時間 + 失敗實驗 + 試產損失 + 良率犧牲 + 人才培訓 + 法規延遲 + 機會成本**

台積電的製程 know-how 需要十幾年的製造經驗才能累積。Dolby 的生態系需要幾十年的標準嵌入和品牌建設。Waymo 的 LiDAR 技術代表了數百名工程師數年的研發。偷竊的本質就是試圖壓縮這些時間。

### 台灣法律特色

台灣的營業秘密保護在全球相對嚴格：
- 營業秘密法同時有**民事 + 刑事責任**
- 國家安全法對「國家核心關鍵技術」加重保護：竊取可處 **5-12 年**
- 企業未盡防止義務，要負**企業刑事責任**
- 台積電 vs TEL 案就是在這個框架下起訴的 — 不只是商業糾紛，而是國家安全案件

## 第八段：結論（Speaker 6，約 1 分鐘）

### 三個模式，一個道理

| 公司 | 策略 | 結果 |
|-----|------|------|
| Dolby | 把 know-how 打包進授權組合 | 每年約 $1.25B 授權收入，約 92.5% 營收來自授權 |
| 台積電 | 把 know-how 留著，賣服務 | 全球最大晶圓代工，年營收 $90B |
| Waymo/Uber | know-how 被偷走 | 和解 $245M，Uber 最終放棄自駕、損失 $3B |

三個故事告訴我們同一件事：

> **秘密不只是資訊。它是可控的重複性。**

控制好了，它就是源源不絕的現金流。
控制不好，法律保護和商業價值可能在一夕之間崩塌。

而控制的方式 — 決策矩陣、授權包設計、BMC 定位、五層內控、稽核機制 — 就是今天報告的全部內容。
