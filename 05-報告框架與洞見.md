# 報告框架與洞見

## 敘事弧線

### 開場 Hook
> 「如果所有人都看得到食譜，為什麼不是每個人都能像台積電、Dolby、Waymo 一樣做出來？」

答案：因為真正值錢的不是資訊本身，而是**可控的重複執行能力** — 調校、良率、除錯經驗、供應商整合、認證流程。

### 核心主張
> 「專利保護公開的邊界。營業秘密保護私有的執行系統。」
> 「營業秘密不只是隱藏的資訊 — 它是對執行力的商業模式控制。」

### 三案三角形
- **Waymo v. Uber** → 秘密洩漏會怎樣
- **台積電** → 最強模式：不授權，留著賣服務
- **Dolby** → 專利 + know-how + 品牌打包 = 持續性現金流

### 結論金句
> 「秘密不只是資訊。它是可控的重複性。」(The secret is not just information. It is controlled repeatability.)

## 8 個讓報告脫穎而出的洞見

1. **最強的營業秘密模式可能是根本不授權** — 台積電留著秘密賣服務
2. **營業秘密竊盜是時間壓縮攻擊** — 偷的人買的是跳過研發學費的速度
3. **專利和秘密是互補的，不是二選一** — 專利保護可見架構，秘密保護操作層
4. **授權是分階段的信任** — 永遠不要第一天就把所有東西交出去
5. **稽核是變現機制的一部分** — Dolby 式合規不只是法律清理，是保護持續性營收
6. **量產 know-how 通常是負面知識** — 「什麼不能做」比最終配方更值錢
7. **供應商是洩漏面** — TSMC/TEL 案證明保護必須涵蓋整條供應鏈
8. **營業秘密需要完整的「作業系統」** — NDA 單獨不夠，需要日誌、權限、浮水印、離職審查、文化執行

## 整體邏輯線

```
秘密識別 → 保護控制 → 分階段揭露 → 授權經濟 → 稽核/執行 → 持續性競爭護城河
```

這條線串起所有五個區塊。教授要求的決策矩陣是起點，但真正的亮點是展示：**你選擇把什麼申請專利、什麼保密，決定了你怎麼定價、怎麼揭露、怎麼稽核、怎麼防禦、怎麼規模化這個技術。**
