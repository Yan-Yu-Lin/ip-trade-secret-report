# 概念基礎：專利 vs 營業秘密

## 核心論點

營業秘密授權不是在「賣資訊」，是在賣 **可控的不對稱優勢** — 更快的良率學習、避免試錯、更低的失敗風險、特定的操作參數。

> 專利保護的是「公開的邊界」(public boundary)
> 營業秘密保護的是「私有的執行系統」(private operating system)

## 法律定義

### 台灣營業秘密法
定義為：方法、技術、製程、配方、程式、設計或其他可用於生產、銷售或經營之資訊，且符合三要件：
1. 非一般涉及該類資訊之人所知
2. 因其秘密性而具有實際或潛在之經濟價值
3. 所有人已採取合理之保密措施

### 美國 DTSA (Defend Trade Secrets Act)
類似定義：廣泛的商業/技術資訊、合理保密措施、因不被一般人知悉或不易被查知而具獨立經濟價值。

## 決策矩陣：什麼時候該專利、什麼時候該保密？

核心問題不是「專利還是秘密？」，而是 **「公開之後會怎樣？」**

| 評估因素 | 傾向申請專利 | 傾向保留為營業秘密 |
|---------|-----------|---------------|
| 逆向工程 | 產品一賣出去就能被拆解分析 | 製程藏在工廠裡，配方/參數外部看不到 |
| 技術生命週期 | 夠長值得申請，但不超過 20 年保護期 | 價值可能超過 20 年（如 Coca-Cola） |
| 侵權偵測 | 從產品、進口紀錄、拆解就能發現侵權 | 濫用發生在被授權方/競爭對手的工廠內，極難舉證 |
| 廣告價值 | 專利能幫助募資、建立客戶信任（「專利技術」品牌效應） | 公開會讓競爭對手學到太多 |
| 獨立發明風險 | 競爭對手可能很快自己發明出來 | 沒有昂貴實驗很難重新發現 |
| 協作需求 | 多家合作夥伴需要明確的註冊權利 | 少數夥伴，可控制的技術移轉 |
| 訴訟舉證 | 專利範圍比對相對容易 | 需要日誌、存取紀錄、NDA、數位鑑識 |

### 混合策略（Hybrid）
**專利保護可見的大架構，營業秘密保護操作層的 know-how。**

以半導體為例：

| 申請專利的部分 | 保留為營業秘密的部分 |
|-------------|-----------------|
| 元件結構（如 GAAFET/奈米片架構） | 精確的製程窗口（溫度、壓力、氣體比例、時序） |
| 廣泛的整合方法 | 針對特定設備的調校配方 (recipe) |
| 設備配置概念 | 哪些缺陷訊號能預測良率崩潰 |
| 量測方法 | 良率學習資料庫 |
| 封裝架構 | 翹曲控制參數、接合順序調校 |

經驗法則：**專利保護競爭對手能觀察到或會獨立收斂到的東西；營業秘密保護只有靠痛苦的製造重複才能學到的東西。**

## 關鍵非對稱性

- 專利擋得住獨立發明（即使對方自己想到的，你有專利就能告），但 **20 年後過期**
- 營業秘密可以永遠持續，但 **擋不住合法的逆向工程或獨立開發**
- 營業秘密一旦洩漏就「瞬間歸零」，不像專利有優雅的到期過程

## 常見誤解

1. 營業秘密不是「我們沒公開它」就算 — 你需要**合理保密措施**
2. 光有 NDA 不夠 — 法院和被授權方看的是實際操作行為
3. 營業秘密法不阻止合法的逆向工程
4. 洩漏的營業秘密不會像專利一樣「優雅過期」— 一旦公開，資產瞬間崩塌
5. 授權如果在太早的階段過度揭露，反而會摧毀這個資產
