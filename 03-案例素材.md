# 案例素材（完整版）

## 案例一：Waymo v. Uber（反面 — 營業秘密竊盜）

### 角色定位
營業秘密竊盜本質上是**時間壓縮攻擊** — 偷的人買的是跳過研發學費的速度。

### 偷了什麼？

Anthony Levandowski 下載了 **14,107 份檔案**，共 **9.7 GB**，內容包含：
- **客製化 LiDAR 硬體設計**：Waymo 自研光達感測器的電路板藍圖（讓自駕車「看見」周圍環境的核心技術）
- **電路板詳細工程圖**：包括當前世代的設計
- **製造細節**：供應商名單、製造規格、技術工作說明書
- **測試文件**：LiDAR 系統安裝和測試指南
- **專案追蹤文件**：「Chauffeur TL weekly updates — Q4 2015」（Google 自駕車計畫內部代號 "Project Chauffeur"）
- **原始碼**：與無人車技術相關的電子設計檔案和程式碼
- 另外還有 **50,000 封 Google 工作信件** 在他的個人電腦上

#### 技術重要性
當時第三方商用 LiDAR 單顆要價超過 **$75,000**。Waymo 的自研設計更便宜且更好 — 這是數百名工程師、數千小時、數百萬美元投資的成果。Waymo 把這次竊盜比喻為「從飲料公司偷走秘密配方」。

### 怎麼偷的？

1. 離職前六週（約 2015/12），Levandowski 在公司筆電上**安裝專用軟體**取得 Waymo 內部設計伺服器的存取權
2. 下載 9.7 GB 檔案到公司筆電
3. 接上 **USB 外接硬碟**，把檔案搬出去
4. **擦掉筆電** — 重灌 Linux 覆蓋 Windows，試圖銷毀數位鑑識證據
5. 另外用**手機拍攝數千張螢幕照片**

### 怎麼被發現的？

**意外的 CC 信件 — 「冒煙的槍」：**
2016/12/13，Waymo 員工被意外 CC 進一封來自 LiDAR 零件供應商的信件：
- 標題：**「OTTO FILES」**
- 原本要寄給 Uber/Otto，收件人包含供應商的「Uber team」信箱
- 附件是 Otto/Uber 的 LiDAR 電路板設計圖
- 設計跟 Waymo 的機密電路板設計**驚人地相似**

這觸發了 Waymo 的內部調查。

**Google 的三個安全工具確認了竊盜：**
- **Armada**：確認下載的網路位址來自 Levandowski 的公司筆電
- **Bit9**：日誌記錄了 USB 硬碟的連接和移除
- **GRR (Google Rapid Response)**：即使筆電被擦掉，GRR 已經把存取日誌傳回 Google 伺服器，可以還原他存取了哪些檔案

### Uber 知道他偷的東西嗎？

**知道。** Uber 收購前請了鑑識公司 **Stroz Friedberg** 做盡職調查，報告揭露：
- Levandowski 離職後仍存取 Google 檔案
- 他個人電腦上有 50,000 封 Google 工作信件
- 他在奧克蘭一間叫 **Shred Works** 的商業碎紙中心銷毀了五張含有 Google 機密的光碟
- 他在鑑識調查進行中嘗試刪除檔案、要人刪 iMessages
- 他甚至**在 Stroz Friedberg 辦公室裡**嘗試清空 Macbook Pro 的垃圾桶
- Uber CEO Travis Kalanick 被告知這些光碟的存在，據稱說「他不想跟這些光碟有任何關係」

**儘管知道這一切，Uber 仍然以 $680M 收購 Otto，並讓 Levandowski 擔任自駕部門主管。** 而且 Stroz 報告**從未提交給 Uber 董事會**。

### 內控失敗分析

| 失敗點 | 說明 |

| 無即時大量下載警報 | 日誌有記錄但沒人即時監看 |
| USB 埠未管制 | 能自由接外接硬碟轉檔 |

| 離職後仍可存取 | 離職後帳號未立即撤銷 |
| 無離職資料稽核 | 未在離職前/後審查員工的資料存取模式 |

## 案例二：台積電晶圓代工模式（正面 — 不授權，靠秘密賣服務）

（此部分內容同先前版本，已包含 TSMC vs TEL 案完整時間線和內控分析。）

### 補充：為什麼「不授權」是最強模式？

台積電的 know-how 不是一個單一配方，而是一整個系統：
- 製程窗口（溫度、壓力、氣體比例、時序）
- 良率學習資料庫（什麼條件下良率會崩潰）
- 設備調校配方（針對每一台機器的最佳參數）
- 缺陷診斷（哪些訊號預示問題）
- 供應商整合（哪個化學品批次會造成變異）
- 客戶協作流程

把這些授權出去等於同時：失去秘密 + 創造競爭對手。所以台積電選擇把秘密留著，賣的是「我幫你做」的服務。

客戶（Apple、Nvidia、AMD 等）付的是代工費，但實際上買的是台積電的秘密製程能力帶來的：
- 更高良率 → 更低單位成本
- 更快投產 → 更早上市
- 更可靠的量產 → 更低風險

## 案例三：Dolby Laboratories（正面 — 專利 + know-how 打包授權）

### 角色定位
不做硬體產品，靠 **專利 + know-how + 軟體 + 品牌 + 認證** 的組合包收持續性授權金。營收的 92.5% 來自授權。

### 授權模型的核心結構：雙層制 (Two-Tier System)

1980 年代發展出的模型：

- **Implementation Licensees（實作授權方）** — 晶片商（如 Qualcomm、MediaTek）把 Dolby 技術整合進晶片，但只能賣給 Dolby 認可的系統授權方
- **System Licensees（系統授權方）** — 設備製造商（Samsung、LG、Apple 等）用這些晶片做終端產品，每台出貨付授權金給 Dolby

Dolby 透過控制「誰能買到有 Dolby 技術的晶片」來確保每台設備都有付錢。

### 授權包裡有什麼？（根據實際 SEC 授權合約）

| 組成 | 內容 |
|-----|------|
| **專利** | 數百項全球專利（音訊壓縮、物件音訊、HDR 影像等） |
| **Know-How** | 「專有資訊，包括但不限於營業秘密、軟體、設計、圖紙、規格」 |
| **Show-How** | 「有價值且必要的教學、技能、經驗、報告、測試、測試結果、教育、技巧和示範」 |
| **受版權保護的作品** | 開發套件中累積的所有受版權保護作品 |
| **商標** | Dolby 名稱和雙 D 標誌 |
| **開發套件 (SDK)** | 產品規格、功能要求、測試訊號、測試程序、Logo 字型、實作手冊 |

### 授權費率

- 按**聲道數**計費：2 聲道 vs 5.1 vs 7.1 費率不同
- **階梯式量價**：每季 10,000 / 50,000 / 250,000 / 1,000,000 聲道為斷點，量越大單價越低
- 實際金額保密，但業界分析指出範圍在**每台設備幾美分到低個位數美元**
- 授權越多技術（Atmos + Vision），每台總費越高
- 每年依美國 CPI 調整
- 在 Dolby 無專利的國家享有折扣

### 認證流程

1. **提交申請** — 提供公司/聯絡/財務/產品資訊
2. **簽約 + 付初始費** — 不是授權金，是「授權與行政設定費」；Dolby 提供 SDK
3. **原型開發** — 按 SDK 規格整合 Dolby 技術
4. **產品測試與核准** — **每個含 Dolby 技術的產品都必須提交測試，獲得 Dolby 核准後才能上市銷售**
   - 測試內容：(a) 與 Dolby 參考產品的效能比對 (b) 比較聽覺測試 (c) SDK 規格合規 (d) Show-How 合規 (e) 影響終端用戶感知的品質標準 (f) 合理的品質和工藝標準
5. 通過後列入 **SQR（合格接收方名單）**，才能向授權晶片商購買有 Dolby 技術的晶片

### 稽核機制

**合約稽核權：**
- 被授權方必須保留**完整帳簿和銷售紀錄 3 年**
- Dolby 可透過**專業註冊會計師或代理人**檢查
- 每年最多兩次，合理通知、營業時間內進行

**少報的代價：**
- 稽核發現差額**超過應付金額 5%** → 構成**重大違約**
- 被授權方須支付**三倍未付授權金** + 月利率 1.5% 利息 + Dolby 的稽核和法律費用全額
- 該次稽核不計入每年兩次的限制

**交叉驗證系統：**
- Dolby 從晶片供應商收季度報告（出了多少 Dolby 晶片給誰）
- 也從系統授權方收季度報告（出了多少台設備）
- 兩組數據比對 → 如果晶片商說出了 100 萬顆給 A 公司，但 A 公司只報 80 萬台，Dolby 會調查

**蓄意少報：**
- Dolby 可通知晶片供應商該授權方**不再具備良好資格** → 直接斷供 Dolby 晶片
- 須賠**三倍授權金** + 利息 + 全額費用

### 為什麼被授權方願意付錢？

**技術上可以不用 Dolby，但商業上幾乎不可能：**

1. **標準綁定** — Dolby Digital (AC-3) 寫進了 DVD 規格、ATSC 數位電視標準、Blu-ray 規格。**要做符合標準的設備就必須有 Dolby 授權。**
2. **消費者期待** — Dolby 品牌是消費電子最知名的品牌之一，消費者把 Dolby 標誌當作品質訊號
3. **內容生態系** — Netflix、Amazon、Disney+、Apple TV+ 全部用 Dolby 格式輸出。你的電視不能解碼 Dolby Atmos/Vision，用戶體驗就會打折
4. **生態系鎖定** — 內容創作者用 Dolby → 平台用 Dolby 輸出 → 消費者期待 Dolby → 製造商必須授權 Dolby → 循環

> 「跟一個標準競爭，就像跟字母表競爭一樣。」

### 實際執法案例

- **Dolby v. Roku (2024-至今)** — Roku 拒絕讓 Dolby 行使合約稽核權，Dolby 以違約、侵害著作權和專利權提告。Dolby 在德國取得 HEVC SEP 初步禁制令。
- 另外也對 **Snap、Acer、Hisense** 提起專利訴訟

### Dolby 的三重 IP 護城河

即使 Dolby 所有專利明天都過期，競爭對手仍然需要：
1. **營業秘密層** — 專有軟體實作（編譯碼，禁止逆向工程）+ know-how + show-how → 無到期日
2. **品牌層** — Dolby 商標和雙 D 標誌 → 無到期日
3. **認證層** — 品質保證系統（每個產品上市前都要通過 Dolby 測試）

專利有效期 20 年，但營業秘密和品牌**沒有到期日**。這才是 Dolby 真正的長期護城河。
