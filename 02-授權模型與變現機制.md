# 授權模型與變現機制

## 核心觀念

Know-how 授權應該設計成 **分階段的產品**，不是一次性的資訊揭露。

> 「Know-how 授權賣的是信心，不是秘密。授權方應該先證明效果，而不是揭露配方。」

## 營收結構（Revenue Stack）

| 付款類型 | 定價的是什麼 |
|--------|----------|
| 前期授權金 (Upfront fee) | 取得機密資料包的權利、展現誠意、回收沈沒研發成本 |
| 里程碑付款 (Milestones) | 證明被授權方能重現實驗室/試產/商業化的效能 |
| 按量/按營收抽成 (Running royalty) | 分享生產/銷售的上行利潤 |
| 最低授權金 (Minimum royalty) | 防止被授權方拿到技術後擱置不用 |
| 技術支援費 (Technical service fee) | 工程師、培訓、問題排除、現場調試 |
| 稽核權/費用 (Audit right/fee) | 驗證範圍、數量、地區、再授權、保密遵守 |

## 分級交付模型（Tiered Disclosure）

這解決了授權的根本矛盾：**買家要先看到秘密才知道值不值得買，但看了秘密就不是秘密了。**

### 五個階段

1. **NDA + 評估包**：高階流程圖、公開專利、去敏感化的測試數據
2. **付費試用期 (Option period)**：有限的配方/範圍、試產協議
3. **簽約授權**：完整 SOP、設備設定、材料規格
4. **驗收里程碑**：進階製程窗口、良率改善參數
5. **商業量產**：故障排除資料庫、缺陷庫、下一代改進

## 解決「驗貨悖論」的實務機制

| 機制 | 運作方式 |
|-----|--------|
| 評估用 NDA | 限定用途、禁止逆向工程、限定殘留知識條款 |
| 去敏感技術文件包 | 展示效能數據、製程範圍、樣品產出，但不給完整配方 |
| 黑箱展示 (Black-box demo) | 授權方執行製程，被授權方只看結果不看參數 |
| 無塵資料室 (Clean-room data room) | 禁止下載、禁止手機、浮水印、存取紀錄、限時檢閱 |
| 專家審查 | 中立技術專家評估可行性，只報告結論不揭露細節 |
| 選擇權協議 (Option agreement) | 被授權方付費取得排他性/評估權，完整揭露前先付錢 |
| 第三方託管 (Escrow) | 秘密存於中立第三方，只在特定條件下釋出（如破產、支援終止） |
| 里程碑式揭露 | 基礎包先給，付款/試產成功/最低量承諾後再給進階內容 |
| 「無污染」條款 | 買方先文件化自己已知的東西，減少日後關於獨立開發的爭議 |

## 真實數字參考（來自 WIPO）

- 非專屬生技授權：淨銷售額的 1-5%（研究試劑/診斷）到 5-10%（治療/疫苗）
- 某銅製程授權案例：每單位價值 0.6% + 台架測試 $500K + 中試開發 $1.5M
- 這些是框架性數字，不是通用規則

## BMC 商業模式圖分析

營業秘密不只是「關鍵資源」— 它滲透 BMC 的每個區塊：

| BMC 區塊 | 營業秘密的角色 |
|---------|------------|
| 關鍵資源 (KR) | 製程配方、隱性 know-how、缺陷/良率數據、供應商參數 |
| 關鍵活動 (KA) | 保護、改良、培訓、稽核、問題排除 |
| 關鍵夥伴 (KP) | 被授權方、OEM、大學、晶圓廠、NDA 下的外包商 |
| 價值主張 (VP) | 更快投產、更高良率、更低廢品、更低失敗風險 |
| 目標客群 (CS) | 缺乏時間、經驗、設備調校或法規 know-how 的企業 |
| 通路 (CH) | 技轉包、技術服務團隊、受控資料室 |
| 客戶關係 (CR) | 長期支援、稽核、改進共享 |
| 收入來源 (RS) | 前期金、里程碑、權利金、支援費、培訓費、排他溢價 |
| 成本結構 (CS) | 安全、文件化、法律、監控、技術支援、訴訟準備 |

### 跟專利型 BMC 的關鍵差異

> **專利 BMC 說：** 「沒有我的許可你不能合法做這件事。」
> **營業秘密 BMC 說：** 「你可以試，但我能讓你更快、更便宜、更少良率損失地做到。」

專利變現的是「排除權」；營業秘密變現的是「執行優勢」。
