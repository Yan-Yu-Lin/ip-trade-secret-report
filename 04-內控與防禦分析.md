# 內控與防禦分析

## 核心洞見

內控不只是防禦措施 — **內控本身就是變現的基礎建設。**

如果你證明不了你有「合理保密措施」，你的秘密在法律上根本不算營業秘密，你就：
- 沒辦法告偷你東西的人
- 沒辦法讓授權合約有法律效力
- 沒辦法為營業秘密估值

## 五層控制架構

| 層級 | 機制 |
|-----|------|
| 法律層 | NDA、發明歸屬條款、保密存續條款、未經同意不得再授權 |
| 員工層 | 角色式存取控制、培訓紀錄、離職面談、歸還/刪除證明 |
| 技術層 | Need-to-know 權限、DLP (資料外洩防護)、浮水印、日誌記錄、禁止批量匯出 |
| 組織層 | 營業秘密登錄冊、揭露審批流程、供應商管控、無塵室協作規則 |
| 證據層 | 不可竄改的日誌、版本控管的檔案、存取歷史、簽署的交接紀錄 |

## 台灣法律脈絡

### 營業秘密法（含刑事責任）
- 台灣的營業秘密法同時有**民事 + 刑事責任**
- 近年修法加重了刑事罰則

### 國家安全法
- 保護「國家核心關鍵技術」
- 第 8 條：核心技術營業秘密犯罪可處 **5-12 年**；意圖用於境外/中國/港澳者可處 **3-10 年**
- 企業除非能證明「已盡最大努力防止」，否則也要負企業刑事責任

### 國家核心關鍵技術清單
- 行政院/國科會認定的技術清單
- 半導體先進製程製造是核心項目
- 被列入清單的技術受國安法層級保護

### 勞基法第 9-1 條（競業禁止）
要件：
1. 有可保護的營業利益
2. 員工確實接觸過營業秘密
3. 合理的範圍/期間/地區
4. 合理的補償

**最長離職後競業禁止期間：2 年**

→ 所以更好的策略不是「禁止員工去工作」，而是「控制並證明濫用」。

### 台灣 vs 美國比較

| 台灣 | 美國 |
|-----|------|
| 營業秘密法：民事 + 刑事 | DTSA：聯邦民事訴因；EEA：刑事竊密/經濟間諜 |
| 國安法加重保護國家核心技術 | 常用 EEA、出口管制、制裁、CFIUS、ITC、禁制令 |
| 智慧財產及商業法院集中審理 | 聯邦法院、ITC、各州 UTSA 主張、DTSA 主張 |
| 企業未盡防止義務 → 企業刑責 | 企業刑責可能，但合規計畫影響起訴/量刑 |
| 核心關切：半導體技術外流 = 經濟安全 | 核心關切：外國經濟間諜 + 跨州/國際商務 |

### 司法實務重點
台灣最高法院 **102 台上 235** 號判決經常被引用：
> 「合理保密措施」不要求完美滴水不漏的保密，但確實要求有真實的分級/存取控制。

白話說：**標籤不夠，要有實際運作的控制機制。**

## 防禦與替代分析

### 護城河 = 對手的替代成本

```
競爭對手繞過你的成本 = 
  研發時間 + 失敗實驗 + 試產損失 + 良率犧牲 
  + 人才招聘/培訓成本 + 法規延遲 + 機會成本
```

### 為什麼製造 know-how 比產品特徵更難防禦？

競爭對手可能知道你的目標產出，但不知道**製程窗口**：溫度、壓力、雜質容忍度、順序、供應商材料等級、清洗間隔、缺陷診斷、「什麼不該嘗試」。

### 量產 know-how 通常是「負面知識」

> 「經過上千次失敗後知道什麼不能做」，往往比最終配方本身更有價值。

這也是為什麼營業秘密的替代成本特別高 — 你不只要重新發明，還要重新經歷所有失敗。

## 策略配套建議（報告必要項目）

| 層面 | 具體措施 |
|-----|--------|
| 合約 | NDA 設計、競業禁止條款（合理範圍）、保密存續條款、稽核權 |
| 技術 | 文件浮水印、伺服器存取權限隔離（Need-to-know）、DLP、下載偵測 |
| 組織 | 離職面談 (Exit Interview)、法律提醒、歸還/刪除認證、營業秘密登錄 |
| 供應商 | 供應商 NDA、受控協作規則、禁止保留資料、定期稽核 |
